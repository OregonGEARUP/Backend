BlockID: GearUp team can edit this. Default is b{some positive number}. This should be unique ID across every block.
This is the "Main Title" of the block. It is short and appears on the header.
Stages appear as a list under block. List of various stages for a block.

StageID: GearUp team can edit this. Default is s{some positive number}. This should be unique ID across every stages.
This is the stage's main title.
Enter image name. Eg: image.png
Is this checkpoints required to move to next question? yes or no or descriptive text.
  

CheckPointID: GearUp team can edit this. Default is cp{some positive number}. This should be unique ID across every checkpoints.
Enter a short title. This will appear below header.
Enter a description for the checkpoint.
Enter URL where users can find more infos.
Enter descriptive text for URL. Eg: if url is https://ebay.com, then text can be "Shop with eBay!"
Does this checkpoint require uses to save anything? If yes, then select what type of checkpoint is this? Radio, Text Field, Checkbox or Date Picker. Else you can skip this step including the instances.
InstancesID: GearUp team can edit this. Default is i{some positive number}. This should be unique ID across every instances.
This is the actual text that appears next to the checkpoints. Usually single letter.
This is an additional text, in smaller and lighter font. Eg: Select if you were or currently in foster care
criteriaID: GearUp team can edit this. Default is c{some positive number}. This should be unique ID across every criteria.
Key. eg: b2_female_s1_cp1_i2
Value. eg: 0 or 1
